検査図面 — 点/矢視を右ペイン1行 + ライブラリ文言整理

実機確認後の調整案。モード切替をヘッダーから右ペインへ移し、 ライブラリは「新規」無効化だけで足りる、という提案の静的プレビュー。

実装前・静的 1280 × 右ペイン 17rem 文言「矢視」は要確認
所見（賛成寄り・1点だけ注意）
  • 右ペイン1行化は良い — 「今選んでいる点に対する操作」なので、指差し状態の横にモードがある方が自然。ヘッダーは保存・戻るに集中できる。
  • 短名「点」は賛成。一方 「矢視」は再混同リスク（以前の「視線記号」誤解）。現場語として残すなら可だが、短名なら 「指」 の方が意味が直結。下に両案を並べた。
  • 未選択時は設定パネルが消えるため、右ペイン先頭に 点/矢視（または点/指）だけ常時表示するのが安全（次の丸数字を置く導線を切らない）。
  • 「編集（改版）」→「編集」 と 注記削除 は賛成。無効な「新規」＋ title ツールチップで十分。文言が増えるほどタッチ操作の視線が散る。
