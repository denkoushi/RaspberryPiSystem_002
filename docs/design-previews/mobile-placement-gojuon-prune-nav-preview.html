配膳スマホ — 五十音＋A–Z で絞る（カーナビ風・ボタン消去）

静的モックのみ。操作はデモ用（クリックで遷移しない）。「次の1文字として続きうるか」で五十音・アルファベット（A–Z）のボタンを残し、 絶対に出てこない文字を非表示（または極薄）にして視線を誘導する UI 案。 想定ルート例: /kiosk/mobile-placement/find-by-name/gojuon

よく使う語 初期（全表示） 「テ」入力後（絞り） 次の1文字だけ大きく 別案・薄表示 ABC 絞り例

プリセット: 現場で頻出する語を並べる（設定・マスタで差し替え想定）

五十音の前に 横スクロール／折返しのチップを置き、口頭で出やすい語を1タップで入力欄に足すイメージ。 X / Y / Z は軸・治具区分などの記号として色分け（モック）。

実装メモ（プレビュー外）: 各入力プレフィックスごとに、辞書（部品名集合）から 次にありうる文字の集合 を計算し、 五十音グリッドと A–Z グリッドの両方に突き合わせて active / pruned / dim を切り替える（Unicode 正規化で英大文字・小文字を揃えるとよい）。 データが大きいときはサーバでプレフィックス trie やインデックスを検討。
