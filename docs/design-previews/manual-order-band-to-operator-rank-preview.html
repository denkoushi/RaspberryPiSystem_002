この画面の使い方（要点）

  • まず資源CDを1つ選ぶ（単一資源で編集）。
  • 帯A/B/C/保留の順に並びを確認。
  • 帯内を微調整して、最後に連番確定。

他資源CDへ進む順番（推奨）

  • 未確定件数が多い資源を先に処理（例: 581）。
  • 納期切迫 item が多い資源を優先。
  • 同じ流れを各資源CDで繰り返し、順次確定。

このプレビューの意図

  • 帯分けは終点ではなく、確定順位作成の前段。
  • 最終成果物は各資源CDの 1..N（オペレーター提示用）。
  • 全体ランキング未成熟でも運用できる。
